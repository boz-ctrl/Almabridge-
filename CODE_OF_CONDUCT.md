# Code of Conduct

AlmaBridge Community Edition is intended to support trusted digital credential infrastructure for education, employment, public-sector and regulated environments. We want the project community to be constructive, professional and respectful.

## Expected Behaviour

Participants are expected to:

- Be respectful and inclusive.
- Give constructive feedback.
- Focus discussion on the issue, evidence and implementation value.
- Respect different levels of technical, policy and institutional experience.
- Avoid sharing confidential, personal or commercially sensitive information.

## Unacceptable Behaviour

Unacceptable behaviour includes:

- Harassment, intimidation or discriminatory language.
- Personal attacks or sustained unconstructive criticism.
- Publishing private information without permission.
- Sharing credentials, keys, passwords or sensitive learner data.
- Deliberate disruption of the project or community.

## Reporting

Concerns can be raised with the project maintainers. Security issues should be handled under [`SECURITY.md`](SECURITY.md), not through public issue threads.

## Scope

This code applies to project repositories, issues, pull requests, discussions, documentation and related community spaces.
