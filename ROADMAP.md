# AlmaBridge Roadmap

This roadmap sets out the intended development direction for AlmaBridge Community Edition and the commercial open-core pathway.

## Phase 1 — Community Foundation

Goal: provide a credible self-hostable proof of concept for trusted digital credential issuing and verification.

Planned capabilities:

- WordPress plugin foundation
- Manual credential issuing
- Learner credential record
- Issuer dashboard
- Public verification page
- QR code verification
- Credential status: active, expired, revoked
- Basic audit log
- Solana transaction reference fields
- Sample credential JSON
- Starter OpenAPI specification
- Installation and developer documentation

## Phase 2 — Production Readiness

Goal: make the platform suitable for institutional pilot deployments.

Planned capabilities:

- Improved role and capability model
- Credential lifecycle management: issue, verify, suspend, revoke, expire, renew
- More robust audit activity logging
- Admin configuration screens
- Template-based credential creation
- Bulk import prototype
- API authentication model
- Improved verification page hardening
- Basic reporting dashboard
- Test suite and CI workflow

## Phase 3 — Pro Edition Boundary

Goal: define the commercial upgrade path for institutions issuing credentials at scale.

Commercial capabilities:

- White-labelling
- Bulk issuing
- Production Moodle integration
- Issuer credit management
- Advanced templates
- Email notification workflows
- Enhanced reporting
- Support-backed deployment

## Phase 4 — Assurance Edition Boundary

Goal: support quality-assured, regulated and multi-institution credential ecosystems.

Commercial capabilities:

- Issuer Trust Registry
- Accreditation metadata
- Jurisdiction tagging
- Regulator dashboard
- Revocation governance workflow
- Assurance evidence packs
- Verification analytics
- ESCO skills and occupation mapping
- Regulated programme reporting

## Phase 5 — Enterprise and Government Boundary

Goal: support national, sector-wide or enterprise trust infrastructure.

Commercial capabilities:

- Multi-tenant deployment
- SSO integration
- Dedicated hosting
- Advanced APIs
- Data warehouse integration
- Sovereign deployment options
- SLA-backed support
- Security assurance and implementation consulting

## Strategic Product Boundary

Community Edition should answer:

> Can AlmaBridge issue and verify credentials?

Commercial editions should answer:

> Can AlmaBridge support regulated, trusted, institution-wide credential infrastructure?
